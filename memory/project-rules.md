# AIvideo 项目规则
> 从用户反馈中继承的项目约定，处理剧本和人设图时应始终遵守。

## 视觉设计规则

### 角色等级视觉
角色等级不通过勋章、肩章或其他军衔类视觉元素表现。等级是制度设定，不需要在角色服装上体现。

### 服装风格
角色服装以现代日常装为主，不需要废土朋克、做旧、破损等末日风格处理。保持干净、现代的日常穿着。

### 提示词格式偏好
人设图提供 2-3 版：
- 版本A：精简必要版（核心特征，中文）
- 版本B：完整创意版（细节+氛围+风格参考，中文）
- 版本C（可选）：特定风格版（中文）

### Negative Prompt
人设图默认不加 negative prompt。特殊情况如需使用，用中文写 negative prompt。

### 人设提示词文件管理
人设提示词.md 只保留最新的提示词，不累积旧版本。
每次处理新剧集时，用最新的提示词覆盖文件内容。
（用户生图后会自行保存资产库，旧提示词无需保留）

## 新增元素规则（重要）
来自用户明确要求（beaste-props-prompts-rule）：

### 凶兽/灵体
- 新增时必须同步生成视觉设计图提示词
- 输出版本A（精简必要版）+ 版本B（完整创意版）

### 重要道具
- 新增时同步生成提示词
- 输出标准版

原则：不能只记录数据库条目而不出图。新出现的视觉元素需配套生图提示词，方便后续制作参考。

## 剧本分析新增任务：角色设定简报（防OOC）
分析剧本时，同步为重要出场角色整理一份设定简报，内容包括：
- 核心性格关键词（2-3个）
- 本集情绪状态
- 与他人的关键关系
- 行为倾向（压力下/对话中的典型反应）
目的：供后续分镜脚本/演员演绎时保持角色一致性，防止AI因缺乏完整上下文产生角色误判导致OOC。

## 剧本检查交互
发现剧本严重问题时暂停处理，先问用户意见再继续。

## 人设图状态管理
- 已生成的图片在数据库中标记为"已生成"
- 人设提示词.md 始终只保留最新一批提示词，不保存历史版本
