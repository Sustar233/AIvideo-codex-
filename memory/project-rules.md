# AIvideo 项目规则
> 从用户反馈中继承的项目约定。当前项目优先服务人设提示词和 AI 视频分镜脚本；剧本分析、数据库维护、场景/道具梳理只作为辅助步骤。

## 视觉设计规则

### 角色等级视觉
角色等级不通过勋章、肩章或其他军衔类视觉元素表现。等级是制度设定，不需要在角色服装上体现。

### 服装风格
角色服装以现代日常装为主，不需要废土朋克、做旧、破损等末日风格处理。保持干净、现代的日常穿着。

### 分镜画面风格
所有分镜脚本的“画面风格”统一固定为：3D写实动漫风格，电影级三维动画质感，次世代CG渲染，高精角色模型，富有层次感的体积光影，电影级景深虚化，无明显2D勾线，虚幻引擎5渲染风格。

除非用户对某一集或某一段分镜明确指定例外，否则不得改写、删减或替换这套基础风格。

### 分镜迭代规则
- 默认支持先出单元预览，再根据反馈继续生成或局部修改。
- 用户反馈只影响相关单元或镜号，不无故重写已确认的剧情节点、台词、站位和镜头设计。
- 剪辑版候补镜头必须与正常版剧情节点、台词节点、关键动作和单元时长对齐。

### 多人场景优化
3 人及以上场景必须启用多人场景优化流程：
- 锁定出场人数、站位、相机轴线和画外人员位置。
- 优先用中近景、越肩、焦点分层和局部裁切降载。
- 接触动作前必须确认角色物理相邻。
- 禁止背景人物无解释消失或前后镜头站位翻转。

### 战斗分镜优化
战斗、追逐、能力释放、凶兽攻击等场景必须启用战斗分镜优化流程：
- 复杂动作拆成起势、蓄力、位移、接触、受力反馈、卸力、结果确认。
- 锁定惯用手、武器、能力源点、攻击方向和受力结果。
- 用空气折射、重力场、电弧、粒子、地表破坏等特效替代爆衣、肌肉膨胀、皮肤开裂。
- 爆炸、碎屑、能量波必须写出三维运动轨迹。

### 提示词格式偏好
人设图提供 2-3 版：
- 版本A：精简必要版（核心特征）
- 版本B：完整创意版（细节+氛围+风格参考）
- 版本C（可选）：特定风格版

### Negative Prompt
人设图默认不加 negative prompt。特殊避坑要求用中文自然语言写入“避坑说明”，不单独输出负面提示词模块。

### 人设提示词文件管理
人设提示词.md 只保留最新的提示词，不累积旧版本。
每次处理新剧集时，用最新的提示词覆盖文件内容。
（用户生图后会自行保存资产库，旧提示词无需保留）

## 新增元素规则（重要）
来自用户明确要求（beaste-props-prompts-rule）：

### 凶兽/灵体
- 新增时必须同步生成视觉设计图提示词
- 输出版本A（精简必要版）+ 版本B（完整创意版）

### 重要道具
- 新增时同步生成提示词
- 输出标准版

原则：不能只记录数据库条目而不出图。新出现的视觉元素需配套生图提示词，方便后续制作参考。

## 剧本检查交互
发现剧本严重问题时暂停处理，先问用户意见再继续。

## 非负责剧集简析规则
- 用户在提交剧本时使用“简单”“简要”“简单解析”等表述，代表该集不是用户负责制作的剧集。
- 此类任务只做简短剧情归纳、关键能力或剧情推进概括，以及必要的明显问题提示。
- 不生成人设提示词、场景提示词、凶兽/灵体/道具视觉提示词，也不展开正式分镜。
- 除非用户另行明确要求，否则不做扩展性资产整理。

## 工作流收束
- 人设任务优先使用 `ai-short-drama-character`。
- 分镜任务优先使用 `storyboard-director`。
- `ScriptAnalyzer` 仅在新剧本导入、数据库更新、或人设/分镜缺少必要设定时做最小分析。
- 不默认生成无关世界观扩写、营销文案、泛化场景图或无关教程。

## 人设图状态管理
- 已生成的图片在数据库中标记为"已生成"
- 人设提示词.md 始终只保留最新一批提示词，不保存历史版本
