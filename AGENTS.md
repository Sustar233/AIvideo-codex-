# AIvideo | 《琥珀之城》人设与分镜专项工作区

项目定位：本仓库以后优先服务两件事：**角色人设图提示词**与**AI 视频分镜脚本**。剧本分析、数据库维护、场景/道具梳理只作为辅助步骤，不再作为默认主流程展开。

## 核心产出
- 人设提示词：`剧本文件夹/人设提示词.md`
- 分镜脚本：`剧本文件夹/第X集_分镜脚本_*.md`
- 剪辑版候补镜头：`剧本文件夹/第X集_*_剪辑版.md`
- 项目数据库：`剧本文件夹/database/`
- 自定义技能：`.agents/skills/`
- 项目记忆：`memory/`

## 默认工作模式
1. **人设优先**：当用户提到角色设计、人设图、人物设定、角色提示词时，优先使用 `ai-short-drama-character`。
2. **日常分镜优先**：普通对白、情绪表演、多人调度、剪辑版和非战斗镜头使用 `storyboard-director`。
3. **战斗专项分流**：只有出现连续对攻、格挡、追逐、摔投、能力交锋或明确战斗设计时，才在 `storyboard-director` 的剧情框架内调用 `seedance-combat-prompt` 设计动作链；单次能力展示、短促推搡或普通走位不加载完整战斗技能。
4. **剧本分析降级为辅助**：只有在用户导入新剧本、明确要求更新数据库、或分镜/人设缺少必要设定时，才调用 `ScriptAnalyzer` 做最小必要提取。
5. **不做无关扩展**：默认不生成营销文案、世界观扩写、非必要场景图、非必要道具图、无关模板或泛化教程。

## 项目世界观锚点
- 城市：琥珀之城（旧称“白鸽之城”），鹰国首都，被琥珀屏障保护。
- 屏障：潘诺西亚外星人的基因时停屏障，黄色蜂窝块碗状形态。
- 制度：末日法典/夕阳法典，人分九等。
- 凶骸：高级凶兽的骨质能源，通过手术赋予人类能力。
- 融骸者：人类最高生态位（凶骸与人体融合）。
- 黄宫：核心治理组织，7名仲裁官。
- 浮梦组织：被琥珀军赶出城的起义军团。

## 主要角色锚点
- 于夜：9级背包客→融骸者→5级仲裁官“地震”，重力异能（魔律克之手）。
- 蓝柚：粉发少女，浮梦战士，凶骸手术者。
- 蔡浩（寸头男）：队长，小人得志。
- 罗岚（鸣蛇）：蛇臂仲裁官。
- 晏江：4级琥珀官，白发军人。
- 赵可（风铃猫）：红发萝莉执行官。
- 完整角色以 `剧本文件夹/database/角色.json` 为准。

## 人设规则
1. 角色等级不通过勋章、肩章视觉化。
2. 角色服装以现代日常装为主，不做废土朋克、脏污做旧或夸张末日装甲。
3. 人设图提示词默认提供 2-3 版：精简版、完整版、可选风格版。
4. 人设图不加 negative prompt；特殊避坑要求用中文自然语言写入说明。
5. `剧本文件夹/人设提示词.md` 只保留最新提示词，不累积旧版本。
6. 新增凶兽、灵体、重要道具时，必须同步生成视觉提示词，但只在它们影响人设或分镜生成时展开。

## 分镜规则
1. 固定画面风格：3D写实动漫风格，电影级三维动画质感，次世代CG渲染，高精角色模型，富有层次感的体积光影，电影级景深虚化，无明显2D勾线，虚幻引擎5渲染风格；除非用户明确指定例外。
2. 每次最多生成 4 个连续单元，不一次性输出整集；用户指定更小范围时按用户范围执行。
3. 新的一集首次制作分镜时只读取一次该集剧本、相关角色资料和场地设定，并建立 `memory/storyboard-cache/第X集.md`；同一集后续单元优先读取缓存及上一批分镜末态，不再重复读取整集资料。剧本、数据库或场地设定发生变化时才刷新缓存。
4. 分镜按快速版、标准版、精细版三档执行。未指定时：非主责集或明确要求快速拆解使用快速版；普通正式制作使用标准版；复杂多人、连续战斗和高风险特效使用精细版。
5. 普通对白和过场镜头只保留一个服务剧情的核心表演动作；完整的眼神、呼吸、眉眼、嘴部和手部变化链只用于重要台词、情绪转折、反应镜和悬念落点。
6. 多人场景必须启用“多人场景优化流程”：人数锁定、站位图谱、轴线锁定、景别降载、画外人员说明、接触动作邻接检查。
7. 战斗场景必须启用“战斗分镜优化流程”：动作切片、力学传导、能力特效绑定、三维粒子轨迹、惯用手/武器锁定、穿帮风险自查。
8. 剪辑版不是时间线表，而是可替换的平行候补分镜；必须与正常版剧情节点、台词节点、关键动作和单元时长对齐。
9. 发现严重剧本问题时暂停，先问用户意见。

## 精简工作流
### 人设
读取角色设定/数据库 → 锁定不可变识别点 → 输出精简版、完整版、可选风格版 → 覆盖写入 `剧本文件夹/人设提示词.md`（如用户要求保存）。

### 分镜
新集首次读取剧本、相关人设和场地设定并建立缓存 → 判断快速/标准/精细模式及普通/多人/战斗/剪辑版类型 → 每批生成不超过 4 个连续单元 → 按场景类型执行分层检查 → 根据用户反馈迭代局部单元。

### 新剧本导入
只做必要审查与资产提取 → 更新新增/变化的人物、凶兽、灵体、重要道具 → 生成必要视觉提示词 → 如需分镜再调用 `storyboard-director`。

## 已安装技能职责
- `ai-short-drama-character`：负责人设图提示词。
- `storyboard-director`：负责剧本单元拆分、普通对白、情绪表演、多人调度、连续性、正式分镜和剪辑版候补镜头。
- `seedance-combat-prompt`：只负责实际战斗单元的招式链、攻防转换、力学传导、战斗运镜与特效轨迹，由 `storyboard-director` 提供剧情边界后调用。
- `ScriptAnalyzer`：仅负责新剧本的最小必要提取、严重问题审查、数据库辅助更新。
- `qiuzhi-skill-creator`：仅当用户明确要求创建/更新技能时使用。
